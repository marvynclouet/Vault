# Priorisation MoSCoW

## Les 4 catégories

### Must have (M)
- Le projet **échoue** sans ça
- Obligatoire pour le lancement
- Exemple : authentification, paiement, fonctionnalité core

### Should have (S)
- Important mais pas vital
- Fort impact sur la satisfaction
- Peut être reporté au Sprint suivant

### Could have (C)
- Nice to have
- Améliore l'expérience
- À faire si le temps le permet

### Won't have (W)
- Explicitement **exclu** du scope actuel
- Évite les malentendus
- Peut être dans une V2

---

## Exemple 1 : App e-commerce mobile
- **M** : Catalogue, panier, paiement, compte utilisateur
- **S** : Recherche, filtres, wishlist, notifications
- **C** : Reviews, comparaison produits, chat support
- **W** : Marketplace vendeurs, AR try-on, abonnement

## Exemple 2 : SaaS B2B dashboard
- **M** : Auth SSO, tableau de bord KPIs, export CSV
- **S** : Alertes, partage de rapports, API
- **C** : Personnalisation graphiques, intégrations
- **W** : White-label, multi-tenant avancé

## Exemple 3 : App mobile fitness
- **M** : Inscription, programmes d'entraînement, suivi séances
- **S** : Intégration Apple Health, défis, partage social
- **C** : Coach vocal, plans nutrition, wearables
- **W** : Réalité augmentée, marketplace équipement
